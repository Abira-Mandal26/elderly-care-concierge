[1:00 - ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's take a look at the system architecture. (pause)
When a user submits a query, it first goes through the Security Checkpoint node. This is a critical pre-processing layer that scrubs sensitive PII, blocks prompt injections, and acts as a dosage guardrail, preventing unauthorized changes to restricted medications like insulin.
If the input is safe, the workflow routes to the Orchestrator Agent. The orchestrator is the central coordinator of our system. It analyzes the user request and dynamically delegates tasks to two specialized sub-agents: the Medication Agent, which tracks daily prescriptions, and the Doctor Visit Coordinator, which manages scheduling.
Both sub-agents connect to a local Model Context Protocol, or MCP, server to securely retrieve or write database records.
If a doctor visit is proposed, the orchestrator routes the flow to a Human-in-the-Loop approval gate node, which pauses execution to confirm the booking before generating the final response.

[3:00 - SECURITY & MCP]
Security is our top priority. Our Security Checkpoint operates entirely on the local system, scrubbing SSNs, credit card details, and medical records before they ever reach the language model. Additionally, we enforce strict guards on restricted drugs: any attempt to modify high-risk dosages is immediately blocked with a warning.
On the integration side, the Model Context Protocol server connects our agents directly to local state databases, exposing custom tools for medication querying, intake logging, and calendar appointments securely.
